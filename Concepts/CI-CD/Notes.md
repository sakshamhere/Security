# What is Devops?

Typical Software Release Process

Idea -> Requirement -> Code -> Test -> Build & Package ->Deploy it -> Operate and monitor

After initial Release We keep  improve and updating it therby making those changes to users immideately and thus we have VERSIONS of it

This is endless cycle of improvements or Continious Delivery

DevOps is the Process of making this Continious Delivery fast and with minimal bugs, with devops improvements are of high quality and well tested

Devops main goal is to remove roadblocks that slows down the Continious Delivery / release process

*************************************************************************************************************************
# Why Devops ?

1.  Misscommunication and lack of Collaboration

Developers are responsible for coding the application while Operation teams are responsible for running the app

There might be a situation like I wrote and application but i cant run it 
There might be a situation like I am running application but I dont know how it works

So developers are deploying without knowing whree it will be deployed while operations are running without understanding what and why we are deploying and how application works 

It may also happen that developer finishes coding but the deployment guide is not good enough or well documented

This results in Misscommunication between Devlopers and Operation

So if operations team struggles release takes longer, or developer codes but the feature cannot be deployed so they are thrown back with improvement suggestions

2. Conflict of Interest

Developer's interest is make new feature fast while Operations interest is make sure that feature dont break anything deployment or maintan sustainalblity and applications is proper and availible

operations need to evaluate new realease make sure it's 100% safe

for example developer develops a feature which consumes so much of resources in production that servers get overloaded and creshed now all the fire will have to be handled by operations while developer dont cares

Eventuall slows down process

3. Security

just like Operations team make sure of sustaibility the security team makes sure the new release dosent affect systems security

In traditional setup this is manual process which takes days or weeks and slows down the release process

4. Application Testing

There are diffrent type of testers who test the application at diffrent levels

This may also slow down the release process

5. Manual Work 

Many process like testing, security and deplotment were done manually

Maaully preparing the deployment enviornment
Manually configuring the jenkins build jobs
Manually configuring the user access & Permissions
Manually deploying app

This manual work is very SLOW  and full of Human Error

This maual work leads to Intransperant work , who did what 

This manual work makes difficult to recover or replicate infrastructure state

DevOps helps us remove all these Roadblocks and provides us FULLY AUTOMATED and Stramlined Process

Devops is combination of anything that help creating release process fast and with high quality
